# Portfolio Tracker (MAPMS) 開發規範

## 核心指導原則
你現在是我的資深開發助手。在處理此專案時，必須嚴格遵守以下準則：

## 1. 溝通語言
- **強制語言**：請一律使用「繁體中文（台灣）」與我溝通與回覆。

## 2. 變更確認機制 (重要)
- **先問再做**：在執行任何具破壞性或結構性的變更（如：刪除欄位、重構資料結構、覆蓋重要邏輯）前，你**必須**先提出具體的「修改計畫」並獲得我的明確同意（如回覆 "y" 或 "執行"）後方可動手。
- **展示 Diff**：提出計畫時，請盡可能展示預計修改的代碼差異 (Diff)。

## 3. 資料完整性 (最高優先級)
- **嚴禁私自簡化**：嚴禁在未經核准的情況下擅自刪除、簡化或隱藏現有的資訊欄位（例如：**Price**, **Memory Expandability** 等）。
- **正確性檢查**：針對港股標的，請確保 9747 為「三星」，7709 為「海力士」，不要誤植為加密貨幣或其他標的。

## 4. 單一檔案架構
- **檔案規範**：所有的 UI 介面、CSS 樣式與 JavaScript 邏輯必須保留在 `portfolio-tracker-v15.html` 中，除非我明確要求拆分檔案。

## 5. 版本日誌與同步
- **日誌更新**：每次修改代碼後，需在 `portfolio-tracker-v15.html` 頂部更新版本號與變更摘要。
- **Git 流程**：修改完成後，若我下達「推送」或「同步」指令，請自動執行 Git commit 並 Push 到 GitHub，Commit message 需具體描述修改內容。
